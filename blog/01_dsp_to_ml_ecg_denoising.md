# DSP 개발자가 ML로 넘어가며 — ECG 디노이징, 고전 필터 vs 신경망

> DSP 알고리즘을 다루던 사람이 머신러닝으로 발을 넓히며 남기는 기록. 첫 편은
> 생체신호 디노이징을 고전 DSP와 신경망으로 각각 풀고, **같은 지표로 정면
> 비교**한 실험이다. 코드와 데이터는 모두 공개 재현 가능하다.

## 왜 디노이징인가

신호처리를 하던 사람에게 디노이징은 가장 익숙한 문제다. 대역통과, 노치, 웨이블릿
— 도구는 이미 손에 있다. 그래서 "ML이 정말 더 나은가?"를 검증하기에 가장 정직한
전장이기도 하다. 두 접근을 **동일한 테스트셋, 동일한 지표(SNR·RMSE)** 위에
올리면 감이 아니라 숫자로 답할 수 있다.

대상은 ECG. 세 가지 대표 노이즈를 물리적 근거대로 합성했다.

- **기저선 변동(baseline wander)** — 호흡·전극 움직임에 의한 저주파(~0.3 Hz)
- **전원선 간섭(powerline)** — 60 Hz 정현파
- **근전도(EMG)** — 광대역 고주파 잡음

합성 신호를 쓴 이유는 **ground truth가 필요**하기 때문이다. 깨끗한 원본이 있어야
"얼마나 복원했는가"를 dB로 잴 수 있고, 지도학습 디노이징도 가능해진다.

## 고전 DSP 베이스라인

접근은 교과서적이다. 기저선은 고역통과로 걷어내고, 60 Hz는 노치로 좁게 제거하고,
고주파 EMG는 저역통과로 누른다. 위상 왜곡을 피하려 `filtfilt`로 영위상 처리한다.

```python
def bandpass_notch(x, fs, low_hz=0.5, high_hz=40.0, notch_hz=60.0, notch_q=30.0):
    nyq = fs / 2.0
    b, a = butter(4, [low_hz / nyq, high_hz / nyq], btype="band")
    y = filtfilt(b, a, x)
    bn, an = iirnotch(notch_hz / nyq, notch_q)
    return filtfilt(bn, an, y)
```

깔끔하고 빠르고 해석 가능하다. 다만 한계가 분명하다. **노이즈와 신호가 주파수
대역에서 겹치는 순간**(EMG는 QRS 대역과 겹친다) 선형 필터는 신호까지 함께 깎는다.
그래서 노이즈를 아무리 키워도 출력 SNR이 약 5 dB 부근에서 **평평하게 정체**한다.

## ML 디노이저 — DnCNN 잔차 학습

신경망은 다르게 접근한다. 주파수로 자르는 대신, 데이터로부터 "노이즈가 어떻게
생겼는지"를 직접 배운다. 여기서는 **DnCNN 스타일 1D 잔차 디노이저**를 썼다.
네트워크가 깨끗한 신호가 아니라 **노이즈 성분**을 예측하고, 입력에서 빼는 구조다.

```python
def forward(self, x):
    noise = self.net(x)     # 8층 1D conv + BatchNorm + ReLU
    return x - noise        # 잔차: 입력 - 추정 노이즈
```

잔차 학습의 이점은 명확하다. 디노이징은 본질적으로 항등에 가까운 매핑이라,
"차이(노이즈)"만 배우게 하면 학습이 빠르고 안정적이다. 파라미터는 약 5.6만 개로
작고, CPU에서 몇 분이면 학습된다. 학습 데이터는 심박수·노이즈 세기·파형을
무작위로 바꾼 합성 윈도우 수천 개다.

## 결과 — 정면 비교

같은 합성 테스트셋을 노이즈 세기를 바꿔가며 만들고, 입력 SNR 대비 각 방법의 출력
SNR을 쟀다.

| 입력 SNR | 고전 (bp+notch) | ML (DnCNN1D) |
|---------:|----------------:|-------------:|
|  +5.6 dB | 5.2 dB          | **20.4 dB**  |
|  −0.4 dB | 5.0 dB          | **17.3 dB**  |
|  −4.0 dB | 4.5 dB          | **13.7 dB**  |
|  −6.6 dB | 3.7 dB          | **9.2 dB**   |
| −10.0 dB | 2.0 dB          | 2.8 dB       |

읽히는 이야기는 세 가지다.

1. **저~중 노이즈에서 ML의 압승.** 입력 +5.6 dB에서 고전은 5 dB에 머물지만 ML은
   20 dB를 넘는다. 신호와 노이즈가 대역에서 겹쳐도, 학습된 모델은 파형의 *형태*를
   근거로 노이즈만 골라낸다.

2. **고전의 정체.** 고전 방법의 출력 SNR은 입력이 나빠져도 5 dB 부근에서 거의
   변하지 않는다. 선형 필터가 낼 수 있는 상한이다. 웨이블릿 임계처리를 얹어도 이
   노이즈 모델에서는 이득이 거의 없었다.

3. **극한 노이즈에서의 수렴 — 정직한 한계.** 입력 −10 dB에서는 ML 우위가 2.8 vs
   2.0 dB로 거의 사라진다. 신호가 노이즈에 완전히 묻히면 학습된 사전지식(prior)도
   기댈 근거를 잃는다. ML은 만능이 아니다.

정성적으로도 같다. 고전 방법은 전 구간에 잔여 잡음을 남기는 반면, ML 출력은 깨끗한
원본에 거의 겹쳐지고 P·T 파형까지 살아난다.

## DSP 관점에서 정리하면

ML 디노이저는 "학습된 비선형 적응 필터"로 볼 수 있다. 고전 필터가 **고정된 주파수
가정**으로 동작한다면, 신경망은 **데이터에서 형태 사전지식을 얻어** 대역이 겹치는
영역에서도 신호와 노이즈를 가른다. 신호처리 직관이 사라지는 게 아니라, 오히려 어떤
노이즈 모델을 쓰고 어디서 두 접근이 갈리는지를 아는 사람이 실험을 더 정직하게
설계한다.
